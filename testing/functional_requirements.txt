7.8.2.7.1.  Register User
================================================================================

Req.ID       : FR-01
Description  : User can create a new account in the system.

  Sys.Req.ID   | System Requirement
  -------------|------------------------------------------------------------
  SR-01        | User requests the registration form.
  SR-02        | User fills in and submits the registration form with
               | Full Name, Email, Password, and Confirm Password.
  SR-03        | The system verifies all the fields and throws error if
               | data cannot be validated.
  SR-04        | The system checks whether the email is unique or not.
  SR-05        | The system shows the message of whether user registration
               | was successful or not.


================================================================================
7.8.2.7.2.  Login User
================================================================================

Req.ID       : FR-02
Description  : Registered user can log into the system with valid credentials.

  Sys.Req.ID   | System Requirement
  -------------|------------------------------------------------------------
  SR-01        | User requests the login form.
  SR-02        | User enters email and password and submits the form.
  SR-03        | The system validates the input fields and throws error
               | if fields are empty or invalid.
  SR-04        | The system authenticates the credentials against the
               | database.
  SR-05        | The system generates a JWT token and redirects the user
               | to the dashboard upon successful login.
  SR-06        | The system displays an error message if credentials
               | are invalid.


================================================================================
7.8.2.7.3.  Reset Password
================================================================================

Req.ID       : FR-03
Description  : User can reset their forgotten password via email verification.

  Sys.Req.ID   | System Requirement
  -------------|------------------------------------------------------------
  SR-01        | User clicks the "Forgot Password" link on the login page.
  SR-02        | User enters their registered email address.
  SR-03        | The system verifies whether the email exists in the
               | database.
  SR-04        | The system sends a password reset link to the registered
               | email address.
  SR-05        | User clicks the reset link and enters a new password.
  SR-06        | The system validates the new password and updates it
               | in the database.
  SR-07        | The system shows a success message and redirects the
               | user to the login page.

7.8.2.7.6.  Create and Manage Trips
================================================================================

Req.ID       : FR-06
Description  : User can create, view, edit, and delete personal trips.

  Sys.Req.ID   | System Requirement
  -------------|------------------------------------------------------------
  SR-01        | User clicks "Create Trip" and fills in trip name,
               | destination, start date, end date, and description.
  SR-02        | The system validates all trip fields including date

               | from the list.
  SR-05        | User can edit existing trip details and save updates.
  SR-06        | User can delete a trip after confirming deletion in the
               | confirmation dialog.
  SR-07        | The system shows a success or error message for each
               | operation performed.

7.8.2.7.7.  Search and Optimize Routes
================================================================================

Req.ID       : FR-07
Description  : User can search for optimized routes between two locations
               on an interactive map.

  Sys.Req.ID   | System Requirement
  -------------|------------------------------------------------------------
  SR-01        | User enters a start location and destination in the
               | Route Optimizer page.
  SR-02        | The system geocodes the locations using the Mapbox API.
  SR-03        | The system calculates and displays the optimized route
               | on the map with distance and estimated duration.
  SR-04        | The system provides turn-by-turn navigation directions.
  SR-05        | The system displays an error message if the location
               | cannot be found.
  SR-06        | User can toggle the real-time traffic layer on the map.

================================================================================

Req.ID       : FR-08
Description  : User can view and filter Points of Interest such as hotels,
               restaurants, and fuel stations along a searched route.

  Sys.Req.ID   | System Requirement
  -------------|------------------------------------------------------------
  SR-01        | The system fetches POIs along the route corridor after
               | a route is calculated.
  SR-02        | POIs are displayed as markers on the map categorized by

               | filters.

               | based on the selected filters.
  SR-05        | User can click on a POI marker to view additional
               | details such as name and category.
