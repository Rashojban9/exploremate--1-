7.8.2.8.2.  Security Requirements
================================================================================

  Req-ID   | Requirement Description                              | Priority
  ---------|------------------------------------------------------|----------
  SR-01    | The security of the server should be managed by the  | Should
           | hosting provider with regular security patches.       |
  ---------|------------------------------------------------------|----------
  SR-02    | The application should not contain any third-party    | Should
           | tracking scripts that affect user experience or       |
           | compromise user privacy.                              |
  ---------|------------------------------------------------------|----------
  SR-03    | The application should use a secure method (BCrypt)   | Should
           | to hash and store user passwords in the database.     |
  ---------|------------------------------------------------------|----------
  SR-04    | Any secured content should be communicated in         | Should
           | encrypted format using HTTPS/TLS protocols.           |
  ---------|------------------------------------------------------|----------
  SR-05    | The application must use JWT (JSON Web Tokens) for    | Should
           | stateless authentication and session management.      |
  ---------|------------------------------------------------------|----------
  SR-06    | The admin panel should be accessible only to users    | Must
           | with the ADMIN role through role-based access         |
           | control (RBAC).                                       |
  ---------|------------------------------------------------------|----------
  SR-07    | API endpoints should validate and sanitize all user   | Should
           | inputs to prevent SQL injection and XSS attacks.      |
  ---------|------------------------------------------------------|----------
  SR-08    | Sensitive API keys (Mapbox, AI service) should be     | Must
           | stored securely in environment variables and never    |
           | exposed to the client-side code.                      |
